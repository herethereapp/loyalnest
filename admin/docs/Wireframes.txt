Explanation of Wireframes
Merchant Dashboard:
- Welcome Page: Setup tasks and congratulatory messages (US-MD1, Phase 1).
- Points Program: Configures earning/redemption rules, branding, and status (US-MD2, Phase 1).
- Referrals Program: Manages SMS/email, social sharing (Phase 2), and referral codes (US-MD3, Phase 1).
- Customers: Lists customers with encrypted email, points, RFM, and search (US-MD4, Phase 1).
- Analytics: Displays metrics, RFM charts, and export options (US-MD5, US-MD13, Phase 1–3).
- Settings: Configures store details, billing, RFM (Phase 2), developer toolkit, and language (Phase 3) (US-MD6, US-MD10, Phase 1–3).
- On-Site Content: Manages loyalty page, rewards panel, launcher, popups, checkout extensions (Phase 2), sticky bar, and post-purchase widget (US-MD7, US-MD16, Phase 1–3).
- VIP Tiers: Configures tiers and perks (US-MD8, Phase 2).
- Activity Logs: Shows points, referrals, VIP changes with filters (US-MD9, Phase 2).
- Bonus Campaigns: Sets up time-sensitive and goal-based campaigns (US-MD12, Phase 3).

Customer Widget:
- Single interface with points balance, redemption, referral popup, RFM nudges (Phase 2), VIP tiers (Phase 2), gamification (Phase 3), and language dropdown (Phase 3) (US-CW1–CW11).
- Includes order cancellation notification (US-CW6) and error states (e.g., insufficient points).

Admin Module:
- Overview: Shows merchant count, points, ROI, and RFM charts (US-AM1, Phase 1).
- Merchants: Lists merchants with points adjustment and suspension (US-AM2, US-AM10, Phase 1).
- Admin Users: Manages admin accounts (US-AM4, Phase 1).
- Logs: Displays API and audit logs (US-AM5, Phase 1).
- Plan Management: Handles plan changes (US-AM7, Phase 2).
- Integration Health: Monitors integrations (US-AM8, Phase 2–3).
- RFM Config Management: Configures RFM thresholds (US-AM9, Phase 2).
- RFM Segment Export: Exports segment data (US-AM11, Phase 3).
- GDPR Requests: Manages data requests/redactions (US-AM6, Phase 1).

Notes:
- ASCII Limitations: Uses simple borders and placeholders ([Chart], [Button: ...]) for clarity. Complex elements are indicated (e.g., [Spinner: Loading]).
- Polaris Compliance: Merchant Dashboard uses Polaris components (Tabs, FormLayout, Button) for Shopify UX standards.
- Tailwind CSS: Customer Widget uses Tailwind for responsive, mobile-friendly design (e.g., [Mobile: Collapsed Widget]).
- Accessibility: Includes ARIA labels (e.g., aria-label="Redeem points") and supports keyboard navigation.
- Scalability: Adds loading states (e.g., [Spinner: Loading Customers]) and export progress bars.
- Security: Includes Shopify OAuth (Merchant Dashboard, Customer Widget) and RBAC (Admin Module) login prompts.
- Multilingualism: Supports language dropdowns and JSONB content (`email_templates.body`, `merchants.language`).
- Analytics: Integrates PostHog tracking (e.g., posthog="redeem_clicked").

How to Use Wireframes:
- Development: Guide React component implementation (e.g., WelcomePage.tsx, Widget.tsx) with Polaris (Merchant Dashboard) and Tailwind (Customer Widget).
- Demo: Share with beta merchants to validate usability or include in Shopify App Store submission.
- Iteration: Test with 1–2 merchants, adjust based on feedback (e.g., simplify redemption options).
- Storage: Save in `docs/wireframes/` as text files or export to PNG via Figma.
